?? Key Learnings from Lavelle's Auto Project

  • Phone Numbers: Always confirm the correct phone number FIRST. We had to update across 10+ files.
  • GHL vs GitHub: Clients don't understand the difference. Need clear explanation that GitHub = code storage, GHL = live hosting.
  • Copy/Paste to GHL: This manual step causes issues. Consider automation or clearer instructions.
  • Navigation Links: Must use full GHL URLs (https://subdomain.exposuresolutions.me/page), not relative paths.
  • Emojis in Code: Can get corrupted. Use Unicode or emoji CDN instead of copy/paste.
  • Page Slugs: GHL generates random slugs for pages (e.g., /home-703153). Need to document these.
  • Header Consistency: Every page needs identical header/nav code. Use includes or components.
  • Feedback Early: Get client approval on name/domain/branding BEFORE building full site.
  • Partner Services: Clearly distinguish "our services" vs "partner referrals" to avoid confusion.
  • Mobile Action Bar: Essential for mobile users - always include Call + WhatsApp buttons.

? Pre-Launch Checklist

  • All phone numbers are correct
  • All navigation links work
  • Logo displays correctly on all pages
  • Mobile responsive on iPhone & Android
  • WhatsApp links open correctly
  • Email links work
  • Booking calendar connected
  • Payment processing tested
  • Google Business Profile updated
  • Social media links work
  • Footer consistent on all pages
  • SEO titles & descriptions set
  • Favicon uploaded
  • SSL certificate active (https)
  • Client has approved all content
  • Backup/export of all code to GitHub
